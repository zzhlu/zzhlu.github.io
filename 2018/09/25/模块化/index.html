模块化

模块化是指将一个复杂的功能或系统进行拆分，形成一系列更单一或独立的功能模块，通过这些模块的再组合又可以形成更多不同的功能或系统，以满足各种场景的应用需求。当项目变得越来越复杂，当代码变得越来越多，更好的管理代码，更好的团队协作，需要模块化。好模块的要求：兼容性、可扩展性、高封装性、易使用性、可移植性、最佳性能。

HTML模块化和BEM

BEM 命名

  • Block: 块（代表了更高级别的抽象、模块或组件）
  • Element: 元素（可选，代表block的后代，用于形成一个完整的Block）
  • Modifier: 修饰（可选，代表block不同的版本、状态）

CSS模块化与LESS

Less是一门CSS预处理语言，它扩充了CSS语言，增加了诸如变量、混合（mixin）、函数等功能，让CSS更易维护、方便制作主题、扩充。

JS模块化

常见js模块化规范

CommonJS

CommonJS是服务器端模块的规范，是第一个模块化规范，NodeJS采用这个规范。根据CommonJS规范，一个单独的文件就是一个模块。加载模块使用require方法，该方法读取一个文件并执行，最后返回文件内部的exports对象。

AMD

AMD 即 Asynchronous Module Definition（异步模块定义），它是一个在浏览器端模块化开发的规范，由于不是JavaScript原生支持，需要使用库函数 RequireJS 。

CMD

CMD 即 Common Module Definition（通用模块定义），CMD 规范是国内发展出来的（阿里），就像 AMD 有个 RequireJS ，CMD 有个浏览器的实现 SeaJS 。
