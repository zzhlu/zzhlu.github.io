Web前端安全

XSS 跨站脚本攻击

XSS 全称( Cross Site Scripting ) 跨站脚本攻击，是Web前端安全中最常见的类型。危害程度大小，依赖业务场景的重要程度。

反射型XSS

攻击脚本通过URL提交到服务端，服务端解析后响应，在响应内容中出现攻击代码，最后浏览器解析执行，也称为非持久型XSS。

存储型XSS

提交的代码会存储在服务端，不管是数据库、内存还是文件系统，下次请求目标页面时不用再提交攻击代码。最典型的例子是通过表单提交攻击脚本到数据库，所有目标用户查看从数据库查询出来的内容时，都会执行攻击代码。存储型XSS的攻击是最隐蔽，也是影响范围最大的，也称为持久型XSS。

DOM XSS

攻击代码并不需要服务器解析响应，靠浏览器端的DOM解析。

常见输入点：

  • document.URL
  • document.location
  • window.name
  • document.cookie
  • 表单项的值

常见输出点：

  • document.write() 直接写入
  • xxx.innerHtml = 直接写入
  • document.create() 修改DOM
  • document.location = URL操作
  • window.open() 打开窗口
  • eval() 直接运行

XSS危害

  • 挂马
  • 盗取用户Cookie
  • 钓鱼攻击
  • 爆发式 Web 2.0 蠕虫
  • 蠕虫式DDoS（分布式拒绝服务）攻击

利用流程：挖掘漏洞 -> 构建攻击脚本 -> 绕过XSS Filter -> 伪装输入点 -> 钓鱼发布 -> 清洗获取数据 -> 产生价值

XSS防范

  • 对输入进行过滤，对输出进行编码。
  • 严格管理Cookie的读写，关键Cookie考虑HttpOnly。
  • 推荐使用UBB来存储富文本。
  • 尽量避免引用不可信的外部js，加载外部js代码必须签订保密协议。
  • 不要通过模板获取和处理用户输入数据（URL、表单、referer等），数据统一由后端处理提供。
  • 慎用eval处理不可信的字符串操作。

CSRF 跨站请求伪造

CSRF（Cross Site Request Forgery） 可以在受害者毫不知情的情况下以受害者名义伪造请求发送给受攻击站点，从而在并未授权的情况下执行在权限保护之下的操作。CSRF 通常针对开源网站、多用户网站、社交网站，攻击者需事先知道攻击目标 URL，配合 XSS 一起攻击。

HTML CSRF攻击

利用 HTML 元素能够设置 src/href 等链接地址发出请求，最普遍的攻击。

通过 JS 动态生成标签对象发起 GET 请求。

通过 POST 请求只能通过 form action 提交方式。

JSON/JSONP 劫持

CSRF防范

增加来路判断，检测 Referer 字段。关键请求尽量增加 Token 效验,且 Token 要满足随机性、一次性、保密性。JSONP 返回内容中避免包含敏感内容。
