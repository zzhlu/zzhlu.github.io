BOM

BOM(Browser Object Model) 指浏览器对象模型，由多个对象组成，其中代表浏览器窗口的 Window 对象是 BOM 的顶层对象，其它对象都是该对象的子对象。

window 对象

window 对象是 BOM 的核心对象，代表浏览器的一个实例。

定义全局变量与在 window 对象上直接定义属性的区别：全局变量不能通过 delete 操作符删除，而直接在 window 对象上定义的属性可以。

window 对象包含以下通用对象：document 、 frames 、 history 、 location 、 navigator 、 screen 。

location 对象

location 对象提供了与当前窗口中加载文档有关的信息及一些导航功能。location 对象既是 window 对象的属性，也是 document 对象的属性，window.location 与 document.location 引用的是同一个对象。

location 对象的属性：

  • hash URL中的hash，返回 # 号后面的字符串。
  • host 返回服务器名称和端口号。
  • hostname 返回服务器名称，不带端口号。
  • href 返回当前页面完整的URL。
  • pathname 返回URL中的目录和文件名。例如 /project/test.html 。
  • port 返回URL中指定的端口号，如果URL中不包含端口号则返回空字符串。
  • protocol 返回页面使用的协议，http或https。
  • search 查询字符串，返回 ? 号后面的所有值。

navigator 对象

navigator 对象包含了浏览器的元信息，其中最常用的是 userAgent 、 language 属性。userAgent 属性记录了浏览器当前的硬件平台、操作系统、渲染引擎、浏览器版本等信息。在客户端，js脚本也可以使用 userAgent 来进行一些特性检测，针对不同环境提供不同的解决方案。现代浏览器中，navigator 对象还提供了 geolocation 、 onLine 、 getBatery 等特性，以供用户判断地址位置、在线状态和电池用量等信息。

history 对象

history 对象用于保存用户上网记录。使用 go() 方法可以切换用户访问历史，back() 和 forward() 可以代替 go() ，back() 返回上一页，forward() 返回下一页。H5规范提供了 pushState 、 replaceState 等方法，可以让浏览器不用刷新页面即可改变历史纪录，通过监听 state 状态值，可以实现无缝内容刷新。

screen 对象

screen 对象中保存着与客户端显示器有关的信息，这些信息一般只用于站点分析。width 属性获取整个屏幕的宽，height 属性获取整个屏幕的高。

常用方法

  • alert("提示信息")
  • confirm("确认信息")
  • prompt("弹出输入框")
  • open("url地址", "_black或_self", "新窗口的大小"）
  • close() 关闭当前的网页。
  • setTimeout(函数, 时间) / clearTimeout(定时器名称) 定时器，只执行一次 / 清除定时器。
  • setInterval(函数, 时间) / clearInterval(定时器名称) 定时器，无限执行 / 清除定时器。
  • encodeURI / encodeURIComponent 转码整个URL（不包括元字符和语义字符） / 转码URL的组成部分（不包括语义字符）。
  • decodeURI / decodeURIComponent 整个URL解码 / URL片段解码。
