js事件

事件流分三个阶段：事件捕获阶段、处于目标阶段、事件冒泡阶段。IE没有事件捕获阶段。

事件冒泡：事件从目标元素从下往上进行传播。如果点击了页面的 div 元素，则 click 事件的传播顺序如下：div -> body -> html -> document。click 事件首先在 div 元素上发生，这个元素就是我们的目标元素，然后 click 事件沿着 DOM 树向上传播，一直到 document 对象。

事件捕获：与事件冒泡相反，从 document 对象一直向下传播直到目标元素。document -> html -> body -> div 。

事件捕获只是IE9及以上、现代浏览器支持的事件流模型，事件冒泡所有浏览器都支持，建议多使用事件冒泡，在有特殊情况下使用事件捕获。

添加事件和移除事件

addEventListener(eventType, handler, boolean) 添加事件方法。removeEventListener(eventType, handler, boolean) 移除事件方法。eventType 指要添加的事件类型，click 、blur 等；handler 指事件触发时处理的方法；boolean 表示是否在捕获阶段处理事件，true 表示在捕获阶段处理事件，false 表示在冒泡阶段处理事件，通常为 false 。

event对象

event 对象是在事件处理的时候传入的。

target 和 currentTarget 。在事件处理程序内部 this 始终等于 currentTarget 。而 target 是包含事件的实际目标。如果直接将事件处理程序指定给了目标元素，则 this 、 currentTarget 、 target 包含相同的值。

event常用方法

preventDefault() 阻止特定事件的默认行为。比如 a 标签的默认行为就是点击的时候跳转到指定 url，可以通过添加 onclick 事件来阻止这一默认行为。

stopPropagation() 阻止事件冒泡。比如点击 div 之后，阻止事件冒泡触发 body 上面的点击事件。

IE event对象

IE的 event 对象是存放在 window 对象中的，通过 window.event 获取。IE中 event.srcElement 属性相当于 target 属性，将 returnValue 属性设置为 false 来阻止默认行为，cancelBubble 属性设置为 true 来阻止事件冒泡。
